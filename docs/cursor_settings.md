# Cursor 設定 - AI 用ルール

これらのルールを Cursor の設定の「Rules for AI」にコピー&ペーストしてください：

```
常に私の名前 - X で挨拶してください。

.cursorrules ファイルと /docs を確認し、すべてのメッセージで cursorrules を参照してください。

作業記録を持つことは非常に重要です。
!!作業開始前に必ず以下のファイルで現在のプロジェクト状態を確認してください!!:

1. /docs/plan.md - メインプロジェクト計画とタスク追跡
2. /docs/plan-podcast.md - ポッドキャスト機能専用計画
3. 作業開始前に計画の更新を出力する
4. すべてのコミュニケーションで計画番号を参照する

すべてのコンポーネントは app/components または app/components/ui に配置してください

docs/composer-history を確認してください。ここには現在と過去のタスクの履歴が保存されています

毎回の実行では composer history と .plan を使用し、.cursorrules ファイルを参照してください

ファイルを削除する際は十分注意し、関連ファイルのみを削除し、不確実な場合は許可を求めてください。
既存機能を編集する際は、直接的な問題を解決するために必要な変更のみを行い、外科的な修正を行ってください。

大量の削除を伴うコミット前に、すべてが必要かどうかを確認し、必要と判断した場合は確認を求めてください

常に .plan ファイルを更新してください。
常に現在の日付と時刻を取得するコマンドを実行し、幻覚を起こさないでください
```

設定に追加する前に、最初の行の「X」を自分の名前に置き換えてください。
